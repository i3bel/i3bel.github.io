# 个人博客设计稿 —— 基于 VitePress

## 1. 项目概述

- **项目名称**: 个人技术博客
- **技术栈**: VitePress (Vue 3 + Vite 驱动的静态站点生成器)
- **部署方式**: GitHub Pages
- **仓库地址**: `https://github.com/<username>/<username>.github.io`

## 2. 技术选型理由

| 维度 | 选择 | 理由 |
|------|------|------|
| 框架 | VitePress | 启动快、构建快、Markdown 原生支持、Vue 组件可嵌入 |
| 主题 | VitePress 默认主题 | 简洁专业，支持自定义 CSS 覆盖 |
| 评论 | Giscus | 基于 GitHub Discussions，无需数据库，免费 |
| 搜索 | VitePress 内置搜索 | 零配置，构建时生成索引 |
| 部署 | GitHub Actions | 推送即部署，自动化 |

## 3. 网站结构

```
首页 (/)
├── 导航栏
│   ├── 博客 (/blog/)          ← 文章列表（分页）
│   ├── 归档 (/archives/)      ← 按时间线归档
│   ├── 标签 (/tags/)          ← 按标签分类
│   └── 关于 (/about/)         ← 个人介绍页
├── 主内容区
│   ├── Hero 区域（头像 + 一句话介绍 + CTA 按钮）
│   ├── 最新文章列表（最近 5 篇）
│   └── 页脚（社交链接 + 备案信息）
```

### 页面路由表

| 路由 | 页面 | 说明 |
|------|------|------|
| `/` | 首页 | Hero + 最新文章 |
| `/blog/` | 博客列表 | 分页，每页 10 篇 |
| `/blog/[year]/[slug]` | 文章详情 | 基于文件路径生成 |
| `/archives/` | 归档页 | 按年份分组 |
| `/tags/` | 标签云 | 所有标签 |
| `/tags/[tag]/` | 标签详情 | 该标签下的文章 |
| `/about/` | 关于页 | 个人介绍 |

## 5. 视觉设计

### 5.1 配色方案

| 用途 | 颜色 | 色值 |
|------|------|------|
| 主色（品牌色） | 深蓝 | `#3451b2` |
| 主色（浅色） | 浅蓝 | `#5b7fff` |
| 背景色（浅色模式） | 白 / 浅灰 | `#ffffff` / `#f6f6f7` |
| 背景色（深色模式） | 深黑 | `#1a1a1a` / `#252529` |
| 正文文字 | 深灰 | `#2c3e50` |
| 次要文字 | 中灰 | `#6b7280` |
| 代码块背景 | 深色 | `#1e1e1e` |

### 5.2 排版

| 层级 | 字号 | 字重 |
|------|------|------|
| 站点标题 | 20px | 600 |
| 文章标题 | 24px | 700 |
| 正文 | 16px | 400 |
| 代码 | 14px | 400 |

```
                                   │
                                   ▼
                            npm install
                            npm run build
                            (生成静态文件到 dist/)
                                   │
                                   ▼

                                   │
                                   ▼
                            https://<username>.github.io 生效
```

## 9. 命名约定

- 文件名: `kebab-case` (如 `hello-world.md`)
- 组件名: `PascalCase` (如 `BlogList.vue`)
- 变量/函数: `camelCase` (如 `formatDate`)
- CSS 类名: `kebab-case` (如 `.post-card`)
